## Why hide a door at all?

The portal began as a tiny dog-triggered dialog beside the blog title. The hidden page grew into a globe and approximate visitor-map experiment; the original passphrase later became a friendlier fruit checkpoint where every offered fruit is a truthful answer. The joke stays optional, and the public blog remains useful without finding it.

## A four-step promise

<ol class="project-storyboard" aria-label="Privacy-safe Dogtor portal journey">
  <li class="project-storyboard-step">
    <h3>Notice the clue</h3>
    <p>A small dog beside the public blog title looks slightly out of place. The page remains complete if a visitor never investigates.</p>
  </li>
  <li class="project-storyboard-step">
    <h3>Choose a fruit</h3>
    <p>Activating the clue opens a keyboard-accessible dialog with four visible choices. Every offered fruit receives truthful feedback; there is no hidden phrase to guess.</p>
  </li>
  <li class="project-storyboard-step">
    <h3>Recover cleanly</h3>
    <p>Escape and the visible close action return focus to the dog. A successful visit survives refresh and back navigation in the same tab, while an undiscovered visit still begins from the public clue.</p>
  </li>
  <li class="project-storyboard-step">
    <h3>Ask before precision</h3>
    <p>The experience can stay approximate. More precise location is requested only after an explicit visitor action, and declining it leaves the journey usable.</p>
  </li>
</ol>
